# 图片上传500错误修复说明

## 问题描述

在"录入错题"功能中，用户上传图片时出现 **500 Internal Server Error**，错误信息为：
```
TypeError: 'numpy.bool' object is not iterable
TypeError: vars() argument must have __dict__ attribute
```

## 根本原因

在 `app/utils/image_processing.py` 的 `detect_correction_marks()` 函数中，返回的字典包含numpy类型的值：
- `numpy.bool_` (布尔值)
- `numpy.float64` (浮点数)
- `numpy.int64` (整数)

在 **Python 3.14 + numpy 2.4.4** 环境下，这些numpy类型在FastAPI的JSON序列化过程中会导致错误，因为：
1. numpy 2.0+ 废弃了 `numpy.bool`，改用 `numpy.bool_`
2. FastAPI的Pydantic在序列化时无法正确处理这些numpy类型

## 修复方案

在 `app/utils/image_processing.py` 第166-179行，将所有返回值显式转换为Python原生类型：

### 修复前
```python
return {
    "has_correction_marks": red_ratio > 0.003,
    "red_pixel_ratio": round(red_ratio, 5),
    "has_handwriting": ink_density > 0.12 or num_labels > 50,
    "ink_density": round(ink_density, 4),
    "num_components": int(num_labels),
}
```

### 修复后
```python
# Convert numpy types to Python native types for JSON serialization
return {
    "has_correction_marks": bool(red_ratio > 0.003),
    "red_pixel_ratio": float(round(red_ratio, 5)),
    "has_handwriting": bool(ink_density > 0.12 or num_labels > 50),
    "ink_density": float(round(ink_density, 4)),
    "num_components": int(num_labels),
}
```

## 修复内容

1. **布尔值转换**: `bool()` - 将numpy布尔类型转换为Python bool
2. **浮点数转换**: `float()` - 将numpy.float64转换为Python float
3. **整数转换**: `int()` - 确保numpy.int64转换为Python int

## 如何使用

修复后，无需重启服务（如果使用了 `reload=True`），直接在前端重新上传图片即可：

1. 打开"错题管理"页面
2. 点击"录入错题"
3. 选择"图片上传"标签
4. 上传图片或粘贴截图
5. 系统将正常识别，不再出现500错误

## 技术说明

这是一个常见的numpy与Python类型兼容性问题，在以下场景中需要特别注意：
- FastAPI/Pydantic的JSON序列化
- 数据库ORM操作
- 任何需要将numpy数组/标量转换为标准Python类型的场景

**最佳实践**: 在函数返回值中，始终将numpy类型显式转换为Python原生类型。

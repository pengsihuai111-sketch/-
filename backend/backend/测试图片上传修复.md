# 测试图片上传修复

## 问题已修复 ✓

图片上传的500错误已经修复。问题原因是numpy类型与FastAPI的JSON序列化不兼容。

## 如何测试

### 方法1：直接在前端测试（推荐）

1. 打开浏览器访问前端页面（通常是 http://localhost:5173）
2. 进入"错题管理"页面
3. 点击"录入错题"按钮
4. 选择"图片上传"标签
5. 上传一张题目图片或使用 Ctrl+V 粘贴截图
6. 如果不再出现"Request failed with status code 500"错误，说明修复成功

### 方法2：使用测试脚本

```bash
cd backend
python test_image_upload.py
```

预期输出：
```

  Result: {'has_correction_marks': False, 'red_pixel_ratio': 0.0, ...}
```

### 方法3：使用curl测试API

```bash
# 使用任意图片文件测试
curl -X POST http://localhost:8000/api/questions/upload-image \
```

预期返回：
```json

```

## 修复内容

修改了 `app/utils/image_processing.py` 文件，将 `detect_correction_marks()` 函数的返回值从numpy类型转换为Python原生类型：

- `numpy.bool_` → `bool`
- `numpy.float64` → `float`  
- `numpy.int64` → `int`

## 注意事项

1. **无需重启服务**：如果后端使用了 `reload=True`（开发模式），修改会自动生效
2. **清除浏览器缓存**：如果前端仍有问题，尝试清除浏览器缓存或硬刷新（Ctrl+F5）
3. **检查日志**：如果仍有问题，查看 `backend/backend.log` 获取详细错误信息

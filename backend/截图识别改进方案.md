# 截图识别准确率改进方案

## 问题分析

用户反馈：题目截图识别经常不完整或不准确

**可能的原因**：
1. 截图范围不够 - 用户截图时没有包含完整题目
2. 图片质量问题 - 截图模糊、分辨率低、光线不好
3. 裁剪不当 - 裁剪时切掉了部分题目内容
4. OCR 识别错误 - 手写体、特殊符号、数学公式识别不准
5. 多题混淆 - 一张图包含多道题，AI 识别时混淆

## 改进方案

### 方案1：添加截图指导提示（推荐，易实现）

**目标**：引导用户正确截图

**实现**：
- 在截图上传区域添加提示文字
- 提供截图示例图片
- 添加"截图技巧"帮助按钮

**提示内容**：
```
📸 截图技巧：
✓ 包含完整题目（题号、题干、选项、图表）
✓ 确保文字清晰可读
✓ 避免倾斜、模糊、反光
✓ 一次只截一道题
✓ 留出适当边距
```

**优点**：
- 实现简单，无需修改识别逻辑
- 直接提升用户截图质量
- 成本低

**缺点**：
- 依赖用户配合
- 不能解决已有的识别问题

---

### 方案2：改进裁剪工具（推荐，中等难度）

**目标**：让用户更精确地选择题目区域

**实现**：
1. 裁剪时显示网格线，帮助对齐
2. 添加"智能边界检测"功能，自动识别题目边界
3. 支持多次调整裁剪区域
4. 添加"撤销"和"重做"功能
5. 显示裁剪区域的尺寸提示

**优点**：
- 提升用户体验
- 减少裁剪错误
- 可视化反馈

**缺点**：
- 需要修改前端裁剪组件
- 开发工作量中等

---

### 方案3：识别后人工校对（推荐，易实现）

**目标**：让用户在录入前检查和修改识别结果

**当前状态**：
- 图片上传模式：已有编辑功能 ✓
- PDF 识别模式：已有编辑功能 ✓

**改进点**：
1. 在识别结果旁边显示原图，方便对照
2. 高亮显示 AI 不确定的部分（低置信度）
3. 添加"识别不完整"标记按钮
4. 提供常见错误的快速修正选项

**优点**：
- 用户可以手动修正错误
- 不依赖 AI 准确率
- 灵活性高

**缺点**：
- 增加用户操作步骤
- 不能自动提升识别准确率

---

### 方案4：优化图片预处理（技术改进）

**目标**：在识别前对图片进行优化处理

**实现**：
1. 自动旋转倾斜的图片
2. 增强对比度和清晰度
3. 去除噪点和背景干扰
4. 二值化处理（黑白化）
5. 自动裁剪白边

**代码位置**：`backend/app/utils/image_processing.py`

**优点**：
- 自动提升图片质量
- 对用户透明
- 提升 OCR 准确率

**缺点**：
- 可能过度处理导致失真
- 需要调试参数
- 增加处理时间

---

### 方案5：多模型识别对比（高级方案）

**目标**：使用多个 AI 模型识别，取最佳结果

**实现**：
1. 同时调用 DeepSeek 和 Doubao（或其他模型）
2. 对比识别结果，选择置信度高的
3. 如果结果差异大，提示用户人工确认

**优点**：
- 提升识别准确率
- 互补不同模型的优缺点

**缺点**：
- 成本翻倍（调用多个 API）
- 处理时间增加
- 实现复杂度高

---

### 方案6：识别质量评分（推荐，中等难度）

**目标**：自动评估识别结果的可信度

**实现**：
1. 计算识别置信度分数（0-100）
2. 检查识别结果的完整性：
   - 是否有题号
   - 题目长度是否合理（太短可能不完整）
   - 是否有明显的截断（句子不完整）
3. 低分结果自动标记"需人工确认"
4. 显示质量评分，让用户知道可信度

**评分规则示例**：
```python
def calculate_quality_score(text, confidence):
    score = confidence  # 基础分：AI 置信度
    
    # 检查题目长度
    if len(text) < 10:
        score -= 20  # 太短，可能不完整
    
    # 检查是否有题号
    if not re.match(r'^\d+[.、)]', text):
        score -= 10
    
    # 检查是否有截断
    if text.endswith(('，', '、', '和', '或')):
        score -= 15  # 句子不完整
    
    # 检查是否有乱码
    if '�' in text or '□' in text:
        score -= 20
    
    return max(0, min(100, score))
```

**优点**：
- 自动识别问题
- 引导用户关注低质量结果
- 可以积累数据优化模型

**缺点**：
- 需要设计评分规则
- 可能误判

---

### 方案7：用户反馈学习（长期方案）

**目标**：从用户修正中学习，持续改进

**实现**：
1. 记录用户对识别结果的修改
2. 统计常见错误类型
3. 分析哪些情况识别率低
4. 针对性优化提示词或预处理

**数据收集**：
- 原始图片
- AI 识别结果
- 用户修正后的结果
- 图片特征（分辨率、清晰度等）

**优点**：
- 持续改进
- 基于真实数据
- 可以发现隐藏问题

**缺点**：
- 需要长期积累
- 涉及数据隐私
- 需要分析工具

## 总结

**最有效的改进**：
1. 用户教育（截图技巧提示）- 预防问题
2. 人工校对（显示原图对照）- 发现问题
3. 质量评分（自动标记可疑结果）- 识别问题

**建议优先级**：
- 🔥 高优先级：方案1（截图提示）+ 方案3（原图对照）
- ⭐ 中优先级：方案6（质量评分）
- 💡 低优先级：方案4（图片预处理）、方案5（多模型）

**预期效果**：
- 短期（1周内）：通过提示和对照，减少 30-50% 的识别错误
- 中期（1月内）：通过质量评分和预处理，再减少 20-30% 的错误
- 长期（3月+）：通过用户反馈学习，持续优化

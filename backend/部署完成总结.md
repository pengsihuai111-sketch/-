## 📋 功能说明

### 改进后的 PDF 识别流程

1. **有文本层的 PDF**：
   - 直接提取文本 → 转 Markdown → DeepSeek 结构化
   - 速度快，准确率高

2. **扫描件 PDF**：
   - 使用 Tesseract OCR 提取文字 → 转 Markdown → DeepSeek 结构化
   - 速度较慢（每页 2-5 秒），但能识别扫描件

3. **包含图片的 PDF**：
   - 提取题目中的图片并保存
   - 图片 URLs 存储在 `question_image_urls` 字段

4. **生成完整错题**：
   - 题目内容（`ai_question_text`）
   - 答案（`ai_answer`）
   - 解析（`ai_solution`）
   - 题目图片（`question_image_urls`）

## 🐛 故障排除

### 问题 1：扫描件 PDF 识别失败

**错误信息**：`Tesseract OCR not available` 或 `TesseractNotFoundError`

**解决方法**：
1. 确认 Tesseract 已安装
2. 检查安装路径是否为 `C:\Program Files\Tesseract-OCR`
3. 如果路径不同，修改 `app/utils/pdf_to_markdown.py` 中的路径配置
4. 重启后端服务

### 问题 2：OCR 识别不出中文

**解决方法**：
1. 确认安装了中文语言包（chi_sim）
2. 运行 `tesseract --list-langs` 检查
3. 如果缺少，手动下载 `chi_sim.traineddata` 放到 tessdata 目录

### 问题 3：识别结果没有答案和解析

**可能原因**：
1. 数据库迁移未执行 → 重新执行迁移脚本
2. 后端服务未重启 → 重启服务
3. LLM 返回格式不正确 → 检查后端日志

### 问题 4：题目图片不显示

**可能原因**：
1. 图片提取失败 → 检查后端日志
2. 图片 URL 格式错误 → 检查 `question_image_urls` 字段内容
3. 前端未处理图片显示 → 需要前端配合更新

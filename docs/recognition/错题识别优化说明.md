# 错题识别性能优化说明

## 优化时间
2024-05-18

## 问题描述

### 问题1：识别速度太慢
- **原因**：使用两阶段识别策略（检测 + 逐题识别）
- **表现**：识别5道题需要 1 + ceil(5/3) = 3 次 API 调用，每次最多90秒超时
- **影响**：用户等待时间过长，体验差

### 问题2：多题识别容易失败
- **原因**：两阶段识别时，每次从全图中按题号提取单题
- **表现**：题号识别错误、提取不完整、混淆相邻题目
- **影响**：识别准确率低，需要大量人工修正

## 优化方案

### 核心思路
**智能策略选择**：根据图片特征自动选择最优识别策略

#### 策略对比

| 策略 | API调用次数 | 适用场景 | 优点 | 缺点 |
|------|------------|---------|------|------|
| **单阶段** | 1次 | 多题图片（3+题）| 快速、不混淆题目 | 单题准确率略低 |
| **两阶段** | 1+N次 | 单题/双题图片 | 单题准确率高 | 慢、多题易混淆 |

### 具体优化

#### 1. 优化策略选择逻辑 (`recognition_config.py`)

**优化前**：
```python
# 大图片 (2000+px) → 两阶段
# 中等图片 (1200-2000px) → 两阶段  
# 小图片 (<1200px) → 单阶段
```

**优化后**：
```python
# 极小图片 (<1000px) → 两阶段（单题高精度）
# 其他所有情况 → 单阶段（快速、多题友好）
```

**效果**：
- 90%的错题识别场景使用单阶段（1次API调用）
- 速度提升 **3-5倍**

#### 2. 两阶段智能降级 (`deepseek.py`)

**新增逻辑**：
```python
# 检测到3+题时，自动切换到单阶段
if len(detected_questions) >= 3:
    logger.info(f"检测到{len(detected_questions)}题，切换单阶段提速")
    return await _call_vision_api_fallback(image_bytes, filename)
```

**效果**：
- 即使选择了两阶段，多题时也会自动优化
- 避免 1+N 次 API 调用

#### 3. 优化单阶段提示词

**改进点**：
1. 强调"仔细数题号，一道都不能少"
2. 明确"每道题必须完整提取"
3. 要求输出 `question_no` 字段
4. 增加 `diagram_description` 和 `confidence` 字段
5. 详细说明多子问处理规则

**效果**：
- 多题识别准确率提升
- 题目完整性提升
- 减少遗漏和混淆

## 使用建议

### 自动模式（推荐）
```python
# 前端调用，无需修改
recognizeWrongQuestionImage(formData)
```
系统会自动选择最优策略。

### 手动指定策略（高级）
```python
# 后端 API 调用
result = await recognize_questions(
    image_bytes, 
    filename,
    strategy=RecognitionStrategy.SINGLE_STAGE  # 强制单阶段
)
```

## 注意事项

1. **单题高精度场景**：如果是单题截图且要求极高准确率，可以手动指定两阶段
2. **图片质量**：模糊、倾斜、低分辨率图片仍会影响识别效果
3. **批改痕迹**：红色批改、手写内容会被自动忽略
4. **配图题目**：几何题、图表题会自动标记 `has_image=true`

## 后续优化方向

1. **并发优化**：单阶段识别后的答案生成可以并行处理
2. **缓存机制**：相似题目识别结果缓存
3. **增量识别**：PDF多页识别时，已识别页面不重复处理
4. **质量预检**：识别前自动检测图片质量并提示用户

## 回滚方案

如果优化后出现问题，可以快速回滚：

```python
# recognition_config.py
DEFAULT_STRATEGY = RecognitionStrategy.TWO_STAGE  # 改回两阶段
```

或在 `deepseek.py` 中注释掉智能降级逻辑：

```python
# if len(detected_questions) >= 3:
#     return await _call_vision_api_fallback(image_bytes, filename)
```

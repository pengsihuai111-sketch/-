# 带批改痕迹错题图片识别功能实现计划

## 1. 背景

当前系统已经支持上传图片、截图或 PDF，通过 AI 大模型识别题目并录入错题。但在真实使用中，用户上传的往往不是干净题目，而是已经批改过的试卷或练习单。这类图片中通常包含：

- 印刷体原题；
- 学生手写解答；
- 红笔批改痕迹；
- 圈画、对勾、叉号、分数；
- 老师批注；
- 多道题混排；
- 拍照倾斜、旋转、阴影、纸张边缘等干扰。

如果直接把整张图片丢给 OCR 或 AI 大模型识别，模型容易把学生解答、红笔批注、分数、圈画内容混进题干，导致识别失败或匹配错误。

因此，本功能的重点不是单纯提高 OCR 准确率，而是建立一套“带批改痕迹错题”的专用识别流程。

---

## 2. 建设目标

### 2.1 核心目标

实现一个新的“带批改痕迹错题识别模式”，用于处理已经批改过的试卷、练习单和错题截图。

系统需要做到：

1. 自动判断上传图片是否存在批改痕迹。
2. 自动进行图片预处理，包括旋转、裁剪、去阴影、增强清晰度。
3. 自动识别并弱化红色批改痕迹。
4. 自动切分一页中的多道题。
5. 每道题单独调用 AI 大模型识别。
6. AI 只提取“印刷体原题”，忽略学生手写、红笔批注、对勾、叉号、分数。
7. 根据识别结果匹配题库中的标准原题。
8. 返回候选题，由用户确认后加入错题本。
9. 如果题库中没有匹配题，则进入“新题录入待确认”流程。

### 2.2 设计原则

不要追求“一键全自动直接入库”，推荐采用：

```text
AI 识别候选结果 + 题库标准题干匹配 + 用户确认兜底
```

最终目标是：

```text
脏图识别靠 AI 理解，标准题干靠题库，最终正确性靠用户确认。
```

---

## 3. 总体流程

推荐流程如下：

```text
用户上传图片 / PDF
→ 保存原图
→ PDF 转高清图片
→ 图片预处理
→ 判断是否存在批改痕迹
→ 去除或弱化红色批改痕迹
→ 检测并切分题目区域
→ 每道题单独调用 AI 大模型
→ 输出结构化题目信息
→ 根据关键词和题干匹配题库
→ 返回候选题
→ 用户确认
→ 加入错题本
```

不推荐流程：

```text
上传整张图片 → 直接调用 AI → 直接输出题目 → 直接加入错题本
```

原因是整页图片中包含太多干扰内容，大模型容易误识别。

## 4. 识别模式设计

### 4.1 普通题目识别模式

适用于：

- 干净题目截图；
- 原版试卷 PDF；
- 没有批改痕迹的图片；
- 没有学生手写答案的题目。

流程：

```text
图片预处理 → AI / OCR 识别 → 结构化题目 → 匹配题库或新题入库
```

### 4.2 带批改痕迹识别模式

适用于：

- 有红笔批改；
- 有学生手写过程；
- 有圈画、对勾、叉号、分数；
- 一页中有多道题；
- 题干和手写内容混杂。

流程：

```text
图片预处理
→ 批改痕迹检测
→ 红色痕迹清洗
→ 题目区域切分
→ 单题 AI 识别
→ 题库相似匹配
→ 用户确认
→ 加入错题本
```

### 4.3 自动判断模式

建议前端默认选择“自动判断”。

系统自动检测以下特征：

- 是否有红色批改痕迹；
- 是否有大量手写内容；
- 是否存在多道题；
- 是否是拍照试卷；
- 是否是图片型 PDF。

如果检测到批改痕迹，则自动进入“带批改痕迹识别模式”。

## 5. 前端实现计划

### 5.1 上传入口改造

在现有错题图片上传页面增加识别模式选择：

```text

```

同时增加说明：

```text
如果图片中有红笔批改、手写过程、圈画、对勾或分数，建议使用“带批改痕迹识别”。
```

前端上传时携带：

```json
{
  "recognition_mode": "auto",
  "use_ai_model": true,
  "remove_correction_marks": true,
  "segment_questions": true,
  "match_question_bank": true

```

字段说明：

| 字段 | 说明 |
|---|---|

| `use_ai_model` | 是否调用 AI 大模型 |
| `remove_correction_marks` | 是否清洗批改痕迹 |

| `match_question_bank` | 是否匹配题库 |

识别后不要直接加入错题本，而是进入确认页。

页面结构建议：

```text
左侧：原始图片 / 裁剪题目图
中间：系统识别出的题目区域
右侧：AI 识别结果 + 题库匹配候选
```

每道题展示：

- 题号；
- 裁剪后的题目图；
- AI 识别题干；
- 题型；
- 知识点；
- 难度；
- 题库候选题；
- 相似度；
- 推荐操作。

操作按钮：

```text
[确认加入错题本]
[换一个匹配题]
[手动搜索题库]
[重新框选]
[作为新题录入]
[忽略此题]
```

### 5.4 手动框选兜底

如果系统自动切题不准确，允许用户手动框选。

流程：

```text
用户点击“重新框选”
→ 在图片上拖拽选择题目区域
→ 系统只识别该区域
→ 返回新的识别结果
```

## 7. AI 大模型调用设计

### 7.1 AI 职责边界

AI 大模型不直接决定最终入库，而是负责生成候选结果。

AI 主要负责：

1. 判断图片类型；
2. 判断是否有批改痕迹；
3. 找出题目区域；
4. 区分原题、学生答案、老师批注；
5. 提取印刷体题干；
6. 输出题型、知识点、难度、关键词；
7. 辅助判断候选题库中最可能匹配的题目。

### 7.2 整页分析 Prompt

```text
你是一个小学数学试卷识别助手。请分析这张试卷图片。

任务：
1. 判断图片中是否包含多道题。
2. 判断图片中是否有红色批改痕迹、学生手写内容、圈画、对勾、叉号、分数、老师批注。
3. 找出每道题的题号和大致区域。
4. 区分印刷体原题、学生手写解答和老师批改痕迹。
5. 不要把红色批改、手写演算、对勾、叉号、圈画、分数计入题目正文。
6. 只输出 JSON，不要输出解释性文字。

JSON 格式：
{
  "image_type": "normal_question | corrected_paper | unknown",
  "has_multiple_questions": true,
  "has_correction_marks": true,
  "has_student_handwriting": true,
  "need_question_segmentation": true,
  "questions": [
    {
      "question_no": "1",
      "bbox": [x1, y1, x2, y2],
      "visible_text_summary": "",
      "confidence": 0.0
    }
  ]
}
```

### 7.3 单题识别 Prompt

```text
你是一个小学数学错题识别助手。请从图片中提取“原始印刷题目”。

要求：
1. 只提取试卷上的印刷体题干。
2. 忽略学生手写答案、红色批改、圈画、对勾、叉号、分数、老师批注。
3. 如果某些文字被遮挡，请标记为 unclear，不要编造。
4. 保留数学符号、分数、单位、图形描述。
5. 判断题型、知识点、难度。
6. 输出 JSON，不要输出解释性文字。

JSON 格式：
{
  "question_no": "",
  "question_text": "",
  "question_type": "fill_blank | choice | calculation | problem_solving",
  "knowledge_points": [],
  "difficulty": "基础 | 中等 | 挑战",
  "keywords": [],
  "has_diagram": false,
  "diagram_description": "",
  "is_complete": true,
  "unclear_parts": [],
  "confidence": 0.0
}
```

### 7.4 题库候选匹配 Prompt

```text
下面是从图片中识别出的错题文本，以及题库中检索出的候选题。
请判断哪一道题最可能是原题。

要求：
1. 优先比较题干中的关键数字、人物、单位、条件关系。
2. 不要只看知识点相同。
3. 如果没有可靠匹配，返回 null。
4. 输出 JSON，不要输出解释性文字。

输入：
{
  "recognized_text": "...",
  "keywords": [],
  "candidates": [
    {
      "question_id": 101,
      "question_text": "...",
      "knowledge_point": "工程问题",
      "difficulty": "中等"
    }
  ]
}

输出：
{
  "matched_question_id": 101,
  "similarity": 0.93,
  "reason": "题干中的关键条件一致",
  "need_manual_confirm": false
}
```

## 8. 图像预处理设计

### 8.1 基础预处理

上传图片后先做：

1. 图片格式统一；
2. PDF 转高清图片，建议 200-300 DPI；
3. 自动旋转校正；
4. 透视矫正；
5. 去阴影；
6. 增强对比度；
7. 裁剪纸张边界；
8. 保存处理结果。

### 8.2 红色批改痕迹清洗

常见批改痕迹是红色，可以使用 HSV 色彩空间检测红色区域。

流程：

```text
原始图片
→ 转 HSV 色彩空间
→ 识别红色像素区域
→ 生成 red_mask
→ 对红色区域做弱化或修复
→ 得到 clean_image
```

伪代码：

```python
def remove_red_marks(image):
    hsv = cv2.cvtColor(image, cv2.COLOR_BGR2HSV)

    lower_red_1 = np.array([0, 70, 50])
    upper_red_1 = np.array([10, 255, 255])
    lower_red_2 = np.array([170, 70, 50])
    upper_red_2 = np.array([180, 255, 255])

    mask1 = cv2.inRange(hsv, lower_red_1, upper_red_1)
    mask2 = cv2.inRange(hsv, lower_red_2, upper_red_2)
    red_mask = mask1 + mask2

    clean_image = cv2.inpaint(image, red_mask, 3, cv2.INPAINT_TELEA)
    return clean_image
```

### 8.3 原图与清洗图同时保留

不要只保存清洗后的图片。

建议保存：

| 图片 | 用途 |
|---|---|
| `original_image_url` | 前端展示、人工核对 |
| `clean_image_url` | AI / OCR 识别 |
| `crop_image_url` | 单题识别与错题记录 |

## 9. 题目区域切分设计

### 9.1 切分目标

避免整页混合识别，将一页试卷拆成多个题目区域。

流程：

```text
整页图片
→ 检测题号
→ 根据题号位置和版面结构切分题块
→ 每道题生成 crop_image
→ 单独识别每道题
```

### 9.2 切分方式

可以结合：

1. AI 大模型识别题号和题目区域；
2. OCR 识别题号，例如 `1.`、`2.`、`3.`、`一、`、`二、`；
3. 版面分析模型检测文本块和图形块；
4. 用户手动框选兜底。

第一版建议使用：

```text
AI 大模型识别 bbox + 用户手动调整
```

题块结构：

```json
{
  "block_id": "p2_q4",
  "page_no": 2,
  "question_no": "4",
  "bbox": [100, 300, 900, 520],
  "crop_image_url": "/uploads/crops/p2_q4.jpg",
  "confidence": 0.88
}
```

## 10. 题库匹配设计

### 10.1 匹配目标

AI 识别结果可能不完整，因此不要把 AI 输出直接作为最终题干。建议将 AI 识别文本作为检索线索，最终尽量匹配题库中的标准原题。

流程：

```text
AI 识别题干
→ 提取关键词
→ 搜索题库候选题
→ 计算文本相似度
→ AI 判断最佳匹配
→ 返回 Top 5
→ 用户确认
```

### 10.2 匹配字段

可使用：

- `question_text`
- `knowledge_point`
- `category`
- `question_type`
- `difficulty`
- `keywords`
- `source_school`
- `source_exam`

### 10.3 匹配算法

第一版：

```text
MySQL FULLTEXT + 关键词匹配 + 编辑距离 / Jaccard 相似度
```

增强版：

```text
Elasticsearch / OpenSearch + BM25
```

高阶版：

```text
Embedding 向量检索 + BM25 混合检索
```

### 10.4 匹配评分公式

```text
match_score =
题干文本相似度 * 0.45
+ 关键词匹配度 * 0.20
+ 知识点匹配度 * 0.15
+ 题型匹配度 * 0.10
+ 难度匹配度 * 0.05
+ 数字条件匹配度 * 0.05
```

### 10.5 自动确认规则

```text
match_score >= 0.90 且 AI confidence >= 0.85：
    默认选中该候选题，但仍展示给用户确认

0.70 <= match_score < 0.90：
    展示候选题，用户确认

match_score < 0.70：
    提示手动搜索题库或作为新题录入
```

## 12. 后端处理流程

### 12.1 上传识别流程

```text
1. 用户上传图片或 PDF
2. 后端保存原始文件
3. 创建 recognition_task 记录
4. 如果是 PDF，转为每页图片
5. 对每页图片做预处理
6. 调用 AI 大模型判断图片类型
7. 如果是带批改痕迹图片，执行红色痕迹清洗
8. 生成 clean_image
9. 调用 AI 大模型识别题目区域
10. 裁剪每个题目区域
11. 对每个题块调用 AI 大模型识别题干
12. 根据识别文本搜索题库候选题
13. 调用 AI 或规则算法判断最佳匹配
14. 保存识别结果
15. 返回前端确认页数据
```

### 12.2 用户确认流程

```text
1. 用户查看识别结果
2. 用户选择匹配题库题目
3. 用户填写错误类型、考试名称、备注
4. 点击确认加入错题本
5. 后端写入 user_wrong_questions
6. 关联原图、裁剪图、识别任务
7. 更新 recognition_block 状态为 confirmed
8. 返回成功结果
```

### 12.3 新题录入流程

如果没有匹配到题库原题：

```text
1. 用户点击“作为新题录入”
2. 系统将 AI 识别题干带入题目录入表单
3. 用户补充答案、解析、知识点、难度
4. 保存到 questions 表
5. 再加入 user_wrong_questions
```


### 13.4 题库匹配失败

处理：

```text
提供“手动搜索题库”和“作为新题录入”两个选项。
```

### 13.5 红色清洗误伤

处理：

```text
前端同时展示原图和清洗图，用户可以切换查看。
```

## 18. 最终建议

本功能的关键不是让 AI 一次性识别整张复杂试卷，而是把问题拆开：

```text
图片预处理解决脏图问题；
题目切分解决多题混排问题；
AI 大模型解决语义理解问题；
题库匹配解决标准题干问题；
用户确认解决最终正确性问题。
```

最终推荐方案：

```text
图片清洗 + AI 大模型结构化识别 + 题库相似匹配 + 人工确认
```

这样既能利用现有 AI 大模型能力，又能避免模型被学生手写和老师批改痕迹干扰，整体稳定性会明显高于“整图直接识别”。

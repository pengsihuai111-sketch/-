# 小升初数学题库管理系统 - 线上部署方案

## 📋 方案概述

将系统部署到云服务器，实现稳定的外网访问。

**推荐配置**：
- **服务器**：阿里云/腾讯云/华为云 ECS
- **配置**：2核4G，40GB硬盘（最低配置）
- **系统**：Ubuntu 22.04 LTS
- **成本**：约 ¥100-200/月

---

## 🎯 部署架构

```
用户浏览器
    ↓
域名 (math-bank.yourdomain.com)
    ↓
Nginx (反向代理 + HTTPS)
    ↓
    ├─→ 前端 (Vue3, 端口 5173)
    └─→ 后端 (FastAPI, 端口 8000)
            ↓
        MySQL 数据库 (端口 3306)
```

### 步骤3：服务器环境配置

#### 3.1 更新系统

```bash
# 更新软件包列表
sudo apt update

# 升级已安装的软件包

```

#### 3.2 安装必要软件

```bash

sudo apt install -y python3.11 python3.11-venv python3-pip

# 安装 Node.js 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 安装 Nginx

# 安装 Git
sudo apt install -y git

# 安装其他依赖
sudo apt install -y build-essential libssl-dev libffi-dev python3-dev
```

#### 3.3 配置 MySQL

```bash
# 启动 MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置
sudo mysql_secure_installation
# 按提示设置：
# - 设置 root 密码（例如：YourStrongPassword123!）
# - 移除匿名用户：Y
# - 禁止 root 远程登录：Y
# - 移除测试数据库：Y
# - 重新加载权限表：Y

# 登录 MySQL
sudo mysql -u root -p

# 创建数据库和用户
CREATE DATABASE question_bank_v4 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'mathbank'@'localhost' IDENTIFIED BY 'MathBank2024!';
GRANT ALL PRIVILEGES ON question_bank_v4.* TO 'mathbank'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 步骤6：配置 Nginx

#### 6.1 创建 Nginx 配置

```bash
# 创建配置文件
sudo nano /etc/nginx/sites-available/math-bank

# 粘贴以下配置
```

**Nginx 配置文件**：

```nginx
server {
    listen 80;
    server_name 你的服务器IP;  # 或者你的域名

    # 前端静态文件
    location / {
        root /var/www/math-bank/math_bank_v4/frontend/dist;
        try_files $uri $uri/ /index.html;
        
        # 缓存静态资源
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # 超时设置（PDF识别需要较长时间）
        proxy_connect_timeout 300s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
    }

    # 上传文件访问
    location /uploads {
        alias /var/www/math-bank/math_bank_v4/backend/uploads;
        autoindex off;
    }

    # 文件上传大小限制
    client_max_body_size 50M;
}
```

#### 6.2 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/math-bank /etc/nginx/sites-enabled/

# 删除默认配置
sudo rm /etc/nginx/sites-enabled/default

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
sudo systemctl enable nginx
```

### 步骤7：配置后端服务（使用 Systemd）

#### 7.1 创建 Systemd 服务

```bash
# 创建服务文件
sudo nano /etc/systemd/system/math-bank-backend.service
```

**服务配置**：

```ini
[Unit]
Description=Math Bank Backend API
After=network.target mysql.service

[Service]
Type=simple
User=root
WorkingDirectory=/var/www/math-bank/math_bank_v4/backend
Environment="PATH=/var/www/math-bank/math_bank_v4/backend/venv/bin"
ExecStart=/var/www/math-bank/math_bank_v4/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 2
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

#### 7.2 启动服务

```bash
# 重新加载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start math-bank-backend

# 设置开机自启
sudo systemctl enable math-bank-backend

# 查看状态
sudo systemctl status math-bank-backend

# 查看日志
sudo journalctl -u math-bank-backend -f
```

### 步骤8：配置 HTTPS（可选但推荐）

#### 8.1 安装 Certbot

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx
```

#### 8.2 获取 SSL 证书

**前提**：你需要有一个域名，并将域名解析到服务器IP

```bash
# 获取证书（自动配置 Nginx）
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

# 按提示输入邮箱
# 选择是否重定向 HTTP 到 HTTPS（推荐选择 2）

# 测试自动续期
sudo certbot renew --dry-run
```

#### 8.3 如果没有域名

可以使用服务器IP访问，但无法配置HTTPS。建议：
1. 购买域名（阿里云/腾讯云，约 ¥10-50/年）
2. 配置域名解析到服务器IP
3. 然后执行上述 HTTPS 配置

## 📊 性能优化

### 1. 后端优化

```bash
# 增加 worker 数量（根据 CPU 核心数）
# 编辑 /etc/systemd/system/math-bank-backend.service
# 修改：--workers 4  （2核建议2-4个worker）

sudo systemctl daemon-reload
sudo systemctl restart math-bank-backend
```

### 2. 数据库优化

```bash
# 编辑 MySQL 配置
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

# 添加以下配置
[mysqld]
max_connections = 200
innodb_buffer_pool_size = 1G
innodb_log_file_size = 256M
query_cache_size = 64M

# 重启 MySQL
sudo systemctl restart mysql
```

### 3. Nginx 优化

```bash
# 编辑 Nginx 配置
sudo nano /etc/nginx/nginx.conf

# 修改 worker_processes
worker_processes auto;

# 添加 gzip 压缩
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css text/xml text/javascript application/json application/javascript application/xml+rss;

# 重启 Nginx
sudo systemctl restart nginx
```

## 🔒 安全加固

### 1. 配置防火墙

```bash
# 安装 UFW
sudo apt install -y ufw

# 允许 SSH
sudo ufw allow 22/tcp

# 允许 HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status
```

### 2. 修改 SSH 端口（可选）

```bash
# 编辑 SSH 配置
sudo nano /etc/ssh/sshd_config

# 修改端口（例如改为 2222）
Port 2222

# 禁用 root 密码登录（推荐使用密钥）
PermitRootLogin prohibit-password

# 重启 SSH
sudo systemctl restart sshd

# 记得在防火墙开放新端口
sudo ufw allow 2222/tcp
```

### 3. 定期备份

```bash
# 创建备份脚本
sudo nano /root/backup.sh
```

**备份脚本**：

```bash
#!/bin/bash
BACKUP_DIR="/root/backups"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u mathbank -pMathBank2024! question_bank_v4 > $BACKUP_DIR/db_$DATE.sql

# 备份上传文件
tar -czf $BACKUP_DIR/uploads_$DATE.tar.gz /var/www/math-bank/math_bank_v4/backend/uploads

# 删除7天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete

echo "Backup completed: $DATE"
```

```bash
# 设置执行权限
sudo chmod +x /root/backup.sh

# 添加定时任务（每天凌晨2点备份）
sudo crontab -e

# 添加以下行
0 2 * * * /root/backup.sh >> /var/log/backup.log 2>&1
```

## 🐛 故障排查

### 问题1：后端服务无法启动

```bash
# 查看详细日志
sudo journalctl -u math-bank-backend -n 100 --no-pager

# 常见原因：
# 1. 端口被占用
sudo lsof -i :8000

# 2. 数据库连接失败
# 检查 .env 文件中的数据库配置

# 3. Python 依赖缺失
source /var/www/math-bank/math_bank_v4/backend/venv/bin/activate
pip install -r requirements.txt
```

### 问题2：前端无法访问

```bash
# 检查 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log

# 检查前端文件是否存在
ls -la /var/www/math-bank/math_bank_v4/frontend/dist

# 重新构建前端
cd /var/www/math-bank/math_bank_v4/frontend
npm run build
```

### 问题3：API 请求失败

```bash
# 检查 Nginx 代理配置
sudo nginx -t

# 查看后端日志
sudo journalctl -u math-bank-backend -f

# 测试后端直接访问
curl http://localhost:8000/api/questions/stats
```

### 问题4：数据库连接失败

```bash
# 检查 MySQL 状态
sudo systemctl status mysql
```

# 错题识别功能修复总结

## 问题描述
用户反馈：错题识别功能不能正常识别

## 根本原因分析

经过代码审查，发现以下问题：

1. **Prompt 语言不匹配**
   - 原 VISION_SYSTEM_PROMPT 使用英文
   - 智谱 AI (GLM) 模型对中文 prompt 响应更好

2. **API 调用参数顺序**
   - 原代码将图片放在 content_parts 前面
   - 某些视觉模型更适合文本在前的顺序

3. **图片质量过低**
   - MAX_IMAGE_DIMENSION = 900（太小）
   - JPEG_QUALITY = 50（压缩过度）
   - 导致文字模糊，识别率下降

4. **日志不足**
   - 缺少详细的 API 调用日志
   - 难以排查问题

## 修复方案

### 1. 优化 VISION_SYSTEM_PROMPT (deepseek.py:189-217)

**修改前**:
```python
VISION_SYSTEM_PROMPT = """You are a math question recognition assistant. Analyze the image and extract all questions.

Rules:
1. Extract each question's text content separately
...
10. **Answer generation**: If the image shows handwritten or printed answers, extract them into the answer field. If no answer is visible, solve the question yourself...
```

**修改后**:
```python
VISION_SYSTEM_PROMPT = """你是一个小学数学题目识别助手。请分析图片并提取所有题目。

要求：
1. 提取每道题的完整文本内容
...
10. **答案生成**：
   - 如果图片中有印刷或手写的答案，提取到 answer 字段
   - 如果没有答案，请根据题目内容（包括图形、表格）自己解答，填写 answer 和 solution
```

**改进点**:
- 全部改为中文，提高模型理解能力
- 明确答案生成要求
- 简化指令结构

### 2. 调整 API 调用顺序 (deepseek.py:119-138)

**修改前**:
```python
content_parts = [
    {
        "type": "image_url",
        "image_url": {"url": f"data:image/jpeg;base64,{base64_image}"},
    },
]
if user_text:
    content_parts.append({"type": "text", "text": user_text})
```

**修改后**:
```python
content_parts = []
if user_text:
    content_parts.append({"type": "text", "text": user_text})

content_parts.append({
    "type": "image_url",
    "image_url": {"url": f"data:image/jpeg;base64,{base64_image}"},
})
```

**改进点**:
- 文本 prompt 放在图片前面
- 符合某些视觉模型的最佳实践

### 3. 提高图片质量参数 (deepseek.py:185-187)

**修改前**:
```python
MAX_IMAGE_DIMENSION = 900  # reduced from 1200
JPEG_QUALITY = 50  # reduced from 65
```

**修改后**:
```python
MAX_IMAGE_DIMENSION = 1200  # increased for better OCR quality
JPEG_QUALITY = 85  # increased for better text recognition
```

**改进点**:
- 提高最大尺寸，保留更多细节
- 提高 JPEG 质量，减少压缩损失
- 显著提升文字识别准确率

### 4. 增强日志记录 (deepseek.py:149-181)

**新增**:
```python
logger.info(f"Calling vision API: {api_url}, model: {model}")
# ... API 调用 ...
if resp.status_code != 200:
    logger.error(f"Vision API error {resp.status_code}: {error_text}")
if not content:
    logger.warning("Vision API returned empty content")
else:
    logger.info(f"Vision API response length: {len(content)} chars")
```

**改进点**:
- 记录 API 调用详情
- 记录响应长度
- 记录错误信息
- 便于问题排查

# 小升初数学题库管理系统 - 服务器配置需求分析

## 📊 项目资源需求分析

### 1. 应用组件资源占用

#### 后端 (FastAPI)
- **代码大小**: 1.5 MB
- **Python依赖**: ~150 MB
- **运行内存**: 
  - 单worker: 150-200 MB
  - 2个worker: 300-400 MB
  - 4个worker: 600-800 MB
- **CPU**: 中等（处理API请求、PDF识别）

#### 前端 (Vue 3)
- **构建后大小**: ~5-10 MB
- **运行内存**: 0 MB（静态文件，由Nginx提供）
- **CPU**: 0（静态文件）

#### 数据库 (MySQL 8.0)
- **安装大小**: ~500 MB
- **运行内存**: 400-800 MB（默认配置）
- **数据存储**: 
  - 初始: 100 MB
  - 1000题: ~50 MB
  - 10000题: ~500 MB
  - 预估: 1-2 GB（含图片）

#### Nginx
- **安装大小**: ~10 MB
- **运行内存**: 10-50 MB
- **CPU**: 极低

#### 系统 (Ubuntu 22.04)
- **基础系统**: 2-3 GB
- **系统运行内存**: 200-300 MB

## 🔧 性能优化建议

### 如果选择了低配置服务器

#### 1. 优化MySQL内存占用

```bash
# 编辑 /etc/mysql/mysql.conf.d/mysqld.cnf
[mysqld]
innodb_buffer_pool_size = 256M  # 降低到256M
max_connections = 50             # 减少最大连接数
```

#### 2. 减少FastAPI worker数量

```bash
# 1核1GB: 1个worker
# 1核2GB: 2个worker
# 2核4GB: 4个worker
uvicorn app.main:app --workers 1
```

#### 3. 启用Swap（应急方案）

```bash
# 创建2GB swap
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久生效
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

⚠️ **注意**: Swap会严重影响性能，仅作应急使用

#### 4. 限制PDF识别并发

```python
# 在 deepseek.py 中
semaphore = asyncio.Semaphore(1)  # 限制为1个并发
```
